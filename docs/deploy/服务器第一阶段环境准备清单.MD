# 服务器第一阶段环境准备清单

## 1. 这份清单的目标

这份清单只做一件事：

先把阿里云 Ubuntu 服务器的基础环境准备好，为后续 `Docker Compose + Nginx` 部署 `freight-agent` 做铺垫。

这一步还不正式部署项目，只完成宿主机层面的准备。

---

## 2. 第一阶段完成后的标准

你完成这一阶段后，服务器应该满足：

1. 可以通过 `MobaXterm` 正常 SSH 登录
2. 数据盘已经挂载到 `/data`
3. Docker 已安装
4. Docker Compose 已可用
5. 项目目录、日志目录、部署目录已创建
6. 安全组规则已经按预期设置

如果这些都完成了，下一步再开始做：

- `Dockerfile`
- `docker-compose.yml`
- `Nginx` 配置
- 项目部署启动

---

## 3. 你现在的正确流程

当前推荐流程是：

1. 先准备服务器环境
2. 再准备部署文件
3. 最后再部署项目

更具体地说：

```text
服务器初始化
  -> 数据盘挂载
  -> Docker / Compose 安装
  -> 目录创建
  -> 安全组确认
  -> 再进入项目容器化和部署
```

---

## 4. 当前阶段不要急着做的事

这一阶段先不要做这些：

1. 不要急着 `docker compose up`
2. 不要先改项目源码
3. 不要先折腾 HTTPS
4. 不要先搞 CI/CD
5. 不要先开放应用端口 `8012`

原因很简单：

- 现在先把宿主机底座搭好
- 后面项目部署和排障会简单很多

---

## 5. 安全组当前建议

当前阶段建议开放：

1. `22/TCP`
2. `80/TCP`
3. `443/TCP`

当前阶段不建议开放：

1. `8012/TCP`

因为后续架构是：

```text
外部访问 -> Nginx -> 应用容器
```

而不是：

```text
外部访问 -> 直接打应用 8012
```

---

## 6. 推荐目录规划

数据盘挂载到：

```text
/data
```

然后使用这套目录：

```text
/data
├── apps
│   └── freight-agent
├── logs
│   └── freight-agent
├── backups
│   └── freight-agent
├── docker
│   └── freight-agent
└── tmp
```

目录职责：

- `/data/apps/freight-agent`
  放项目代码

- `/data/logs/freight-agent`
  放运行日志

- `/data/backups/freight-agent`
  放备份

- `/data/docker/freight-agent`
  放部署层文件，例如 `docker-compose.yml`、Nginx 配置、生产环境变量

---

## 7. 第一阶段执行步骤

### 步骤 1：用 MobaXterm 登录服务器

目标：

- 确认能 SSH 登录

连接信息：

- 公网 IP：`47.101.141.117`
- 用户名：你自己确认填写
- 密码：你自己手动输入

首次登录后执行：

```bash
whoami
uname -a
lsblk
df -h
```

确认：

1. 当前登录用户是谁
2. Ubuntu 正常
3. 系统盘和数据盘已识别

### 步骤 2：检查并挂载数据盘

目标：

- 把 1TB 数据盘挂载到 `/data`

常用命令：

```bash
lsblk
sudo fdisk -l
```

如果数据盘还未格式化且是空盘，才执行格式化。  
注意：格式化会清空数据。

示例：

```bash
sudo mkfs.ext4 /dev/vdb
sudo mkdir -p /data
sudo mount /dev/vdb /data
df -h
```

然后配置 `/etc/fstab` 做开机自动挂载。

### 步骤 3：更新系统

目标：

- 把基础系统包更新好

执行：

```bash
sudo apt update
sudo apt upgrade -y
```

### 步骤 4：安装基础工具

目标：

- 为后续部署准备常用命令工具

执行：

```bash
sudo apt install -y curl wget git unzip vim htop tree ca-certificates gnupg lsb-release
```

### 步骤 5：安装 Docker 与 Compose

目标：

- 宿主机具备容器部署能力

你可以直接执行后续提供的脚本，也可以手动装。

安装完成后需要验证：

```bash
docker --version
docker compose version
```

### 步骤 6：创建标准目录

目标：

- 把代码、日志、部署层分开放好

执行：

```bash
sudo mkdir -p /data/apps/freight-agent
sudo mkdir -p /data/logs/freight-agent
sudo mkdir -p /data/backups/freight-agent
sudo mkdir -p /data/docker/freight-agent
sudo mkdir -p /data/tmp
```

如果你不是 root 用户部署，还要改目录权限。

### 步骤 7：执行宿主机初始化脚本

目标：

- 一次性把 Docker 环境和标准目录准备好

脚本文件：

[bootstrap_aliyun_docker_host.sh](D:\CompanyPlace\AIProject\AiFreightRate\freight-agent\docs\deploy\bootstrap_aliyun_docker_host.sh)

建议流程：

1. 先把脚本上传到服务器
2. 登录后执行：

```bash
bash /root/bootstrap_aliyun_docker_host.sh
```

如果你不是 `root`，请把脚本放到你实际用户目录，再执行对应路径。

### 步骤 8：确认第一阶段是否完成

执行这些命令检查：

```bash
docker --version
docker compose version
ls -lah /data
ls -lah /data/apps
ls -lah /data/logs
ls -lah /data/docker
```

你应该看到：

1. Docker 正常
2. Compose 正常
3. `/data/apps/freight-agent` 存在
4. `/data/logs/freight-agent` 存在
5. `/data/docker/freight-agent` 存在

## 8. 第一阶段完成后，下一步做什么

当这一阶段完成后，下一步才进入：

1. 上传或拉取项目代码
2. 补 `Dockerfile`
3. 补 `.dockerignore`
4. 补 `docker-compose.yml`
5. 补 Nginx 配置
6. 准备 `.env.production`
7. 最后 `docker compose up -d --build`

---

## 9. 当前阶段的核心判断

你现在的思路是对的：

- 先搭服务器环境
- 再部署项目

这是最合理的顺序。

因为这样你后续排障会非常清晰：

- Docker 有问题，就是宿主机问题
- Nginx 有问题，就是代理层问题
- Compose 有问题，就是部署编排问题
- 应用有问题，才是项目本身问题

---

## 10. 你现在最应该做的事

如果只看当前阶段，你现在最应该做的是：

1. 登录服务器
2. 挂载数据盘
3. 跑初始化脚本
4. 确认 Docker 和目录都正常

做到这里就够了。  
后面的项目容器化和正式部署，等这一层准备好再继续。
