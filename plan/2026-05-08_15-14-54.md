# AI询价结果展示字段映射调整记录

时间：2026-05-08 15:14:54

## 背景

在 AI 询价查询完成后，返回结果会被整理成 Markdown table 展示。

原实现中：

1. `航线` 列通过代码自行拼接
2. `包装` 列直接使用 `packageType`

但根据最新确认的接口口径：

1. `航线` 列应直接使用接口返回的 `routingDisplay`
2. `包装` 列应直接使用接口返回的 `packingDisplay`

本次改动明确说明：这里指的是后端 AI 询价结果组织逻辑，不是 `logistics.html` 的前端 Markdown 渲染。

## 目标

统一调整 AI 询价结果 Markdown table 的字段来源：

- 航线列：改为 `routingDisplay`
- 包装列：改为 `packingDisplay`

并保持以下原则：

1. 只改展示字段来源
2. 不改查询逻辑
3. 不改上下文/会话逻辑
4. 不改 `logistics.html`
5. 精确结果和类似运价结果统一使用相同规则

## 涉及代码点

文件：

- `graph/nodes.py`

函数：

- `_build_markdown_table`

## 修改前

Markdown table 中两列来源为：

- `航线`：代码拼接 `sfg/zzg/mdg`
- `包装`：`packageType`

## 修改后

Markdown table 中两列来源改为：

- `航线`：`routingDisplay`
- `包装`：`packingDisplay`

## 修改范围

仅影响 AI 询价结果表格中的列值来源，包括：

1. 精确查询结果表格
2. 类似运价结果表格

不影响：

1. 运价查询参数抽取
2. 询价会话状态流转
3. 类似运价降级策略
4. 前端 Markdown 渲染逻辑

## 风险说明

按照已确认要求，本次不加兜底优先级。

因此如果接口某条数据缺失：

- `routingDisplay`
- `packingDisplay`

则对应单元格会显示 `-`。

这符合当前“只统一切字段，不加展示优先级”的口径。

## 验证重点

1. 精确结果表格中：
   - 航线列是否显示 `routingDisplay`
   - 包装列是否显示 `packingDisplay`

2. 类似运价表格中：
   - 航线列是否显示 `routingDisplay`
   - 包装列是否显示 `packingDisplay`

3. 表格结构本身是否保持不变

## 结论

本次改动属于“结果展示字段映射修正”，范围小、风险低，且已按统一口径收敛到单一结果构造函数中。
